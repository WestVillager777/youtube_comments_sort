# youtube_comments_sort

## 1. 概要
YouTube公式APIを使用し、特定の動画内のコメントを「いいね数順（降順）」で並び替えて一覧表示するデスクトップアプリケーションです。
標準機能にはない「高評価コメントの抽出」に特化することで、動画に対する良質なフィードバックや有益な情報を効率的に閲覧することを可能にしました。

## 2. 開発背景・目的（Why） 
**課題:** YouTube標準のコメント欄では、最新順か関連順しか選べず、数千件規模のコメントの中からいいね数の多いコメントを探し出すのが困難。  
**解決:** YouTubeAPIを使って、コメントといいね数の取得をして、ソートすることでこの問題を解決。また、大量データ取得中の「アプリのフリーズ」を防ぐため、並行処理を導入し実用性を高めました。

| **言語** | Python 3.x |

## 4. 技術的なこだわり・工夫（How）

### ① マルチスレッドによる「使いやすいUI」の実現
当初、大量のコメント取得中にGUIがフリーズ（応答なし状態）する問題が発生した。これを解決するため、`threading`モジュールを導入。
「データ取得処理」をサブスレッドへ切り出し、メインスレッド（GUI）を解放することで、取得中もウィンドウの移動や進捗表示（`root.after`を使用）を可能にする「スレッドセーフ」な設計を実装した。

### ② 再帰関数とNextPageTokenによる大規模データ取得
YouTube APIの1リクエストあたりの取得制限を克服するため、`nextPageToken`を用いた再帰的なページネーション処理を実装した。これにより、数千件規模のコメントでも漏れなく取得することができるようになった。

### ③ インクリメンタルな進捗フィードバック
ユーザーが「現在どの程度処理が進んでいるか」を把握できるよう、取得済みの件数をリアルタイムでテキストボックスに出力する機能を搭載。長時間の処理におけるユーザーの不安を解消するUX設計を意識しました。

### ④　閾値の設置により、コメント取得の高速化
全ての良いね数のコメントを取得するとなると時間がかかってしまうので、GUIのなかにいいね数の制限の入力欄を導入することで、1000いいね以上のコメントだけを取得することで、コメント取得の高速化の実装ができた。

## 5. 学習したこと・解決した課題
**並行処理の基礎：** メインスレッドとサブスレッドの役割分担、およびGUIへの安全な干渉方法の習得。  
**再帰ロジックの実装：** 終了条件を明確にした再帰関数による、階層構造データのフラット化。再帰関数を使う際は、無限ループに陥てしまい、APIの無駄な消費などの原因にもつながるため、終了条件を特に細心の注意を払って設定した。
